# Твой PostgreSQL — это болид F1. Хватит ездить на первой передаче

Medium-серия из 4 частей о производительности PostgreSQL для Java-разработчиков. Всё на практике, всё своими руками.

## План серии

### Часть 1. Введение и настройка окружения
- Введение: приложение, отлично работающее на нескольких тысячах пользователей, может сильно деградировать при 1 млн — проблема чаще всего в БД
- Практический подход: не голая теория, а всё своими руками
- Установка PostgreSQL через Docker
- Структура тестовой БД: таблицы, связи, объёмы данных, учебные особенности схемы
- Скрипты для наполнения данными

### Серия «Способы доступа к данным» (части 2-5)

### Часть 2. EXPLAIN ANALYZE и Seq Scan
- Как читать `EXPLAIN` и `EXPLAIN ANALYZE` — учимся понимать план запроса
- **Seq Scan** — когда читаем всё подряд и почему это медленно
- Порог переключения Bitmap → Seq Scan (1, 2, 3 валюты)
- Эксперимент с `random_page_cost`: Seq Scan → Index Scan на лету
- Последовательный vs случайный доступ

### Часть 3. Index Scan и Index Only Scan
- **Index Scan**: B-tree, сравнение с Seq Scan (×216 и ×882)
- Составные индексы и leftmost prefix
- **Index Only Scan**: покрывающий индекс, INCLUDE, Visibility Map
- Эксперимент: UPDATE → Heap Fetches: 70 → VACUUM → 0

### Часть 4. Bitmap Scan и типы индексов
- **Bitmap Heap Scan**: двухфазный подход, BitmapOr
- **Lossy Pages**: work_mem = 64KB vs 4MB
- Обзор типов: Hash (UUID), GIN (JSONB), BRIN (временные ряды)
- BRIN: 56 KB vs 107 MB (×1 955)

### Часть 5. Антипаттерны и Width
- Три случая, когда B-tree не помогает: LIKE, функции, JSONB
- Когда НЕ использовать GIN и функциональные индексы
- **Width**: анатомия строки, SELECT * vs колонки
- Таблица размеров типов данных
- Советы пилота: 8 правил

### Серия «JOIN» (в будущем)

### Часть 3 (JOIN). Способы JOIN-ов таблиц
- **Nested Loop Join** — для маленьких выборок
- **Hash Join** — хеш-таблица в памяти
- **Merge Join** — когда оба набора отсортированы
- Сравнение JOIN по разным типам внешних ключей:
  - `VARCHAR` (строковый)
  - `UUID`
  - `BIGSERIAL` (числовой)
- Влияние типа ключа на размер индекса и скорость JOIN

### Часть 4 (Hibernate). Hibernate и производительность (в будущем)
- N+1 проблема и как её видеть
- FetchType: LAZY vs EAGER
- Batch fetching
- DTO projections
- Кэш первого и второго уровня
