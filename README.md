# Твой PostgreSQL — это болид F1. Хватит ездить на первой передаче

Medium-серия из 4 частей о производительности PostgreSQL для Java-разработчиков. Всё на практике, всё своими руками.

## План серии

### Часть 1. Введение и настройка окружения
- Введение: приложение, отлично работающее на нескольких тысячах пользователей, может сильно деградировать при 1 млн — проблема чаще всего в БД
- Практический подход: не голая теория, а всё своими руками
- Установка PostgreSQL через Docker
- Структура тестовой БД: таблицы, связи, объёмы данных, учебные особенности схемы
- Скрипты для наполнения данными

### Часть 2. Способы доступа к данным
- Как читать `EXPLAIN` и `EXPLAIN ANALYZE` — учимся понимать план запроса
- **Seq Scan** — когда читаем всё подряд и почему это медленно
- **Bitmap Heap Scan** — промежуточный вариант
- **Index Scan** — ускорение через индекс
- **Index Only Scan** — покрывающий индекс, когда в таблицу вообще не ходим
- Каждый способ — на живом примере с таймингами
- Визуализация разницы в скорости

### Часть 3. Способы JOIN-ов таблиц
- **Nested Loop Join** — для маленьких выборок
- **Hash Join** — хеш-таблица в памяти
- **Merge Join** — когда оба набора отсортированы
- Сравнение JOIN по разным типам внешних ключей:
  - `VARCHAR` (строковый)
  - `UUID`
  - `BIGSERIAL` (числовой)
- Влияние типа ключа на размер индекса и скорость JOIN

### Часть 4. Hibernate и производительность (в будущем)
- N+1 проблема и как её видеть
- FetchType: LAZY vs EAGER
- Batch fetching
- DTO projections
- Кэш первого и второго уровня
